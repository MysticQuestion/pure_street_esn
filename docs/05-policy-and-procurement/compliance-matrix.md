# Compliance Matrix

The following matrix summarizes how ESN complies with key legal, accessibility, and procurement requirements.  Use this matrix in procurement responses and grant applications.

| Requirement                      | Description                                                | ESN Compliance                                    |
|---------------------------------|------------------------------------------------------------|---------------------------------------------------|
| **Accessibility (WCAG 2.1 AA)** | Web/mobile interfaces must meet accessibility standards.   | Interfaces designed to meet WCAG 2.1 AA; includes high contrast, keyboard navigation, screen reader support. |
| **Data Privacy (GDPR/CCPA)**    | Personal data must be minimized, secured, and deletable.   | PII minimization, encryption at rest and in transit, user consent, retention policies. |
| **Security**                    | Systems should be protected from unauthorized access.      | TLS for all communications, RBAC, audit logging, vulnerability disclosure process. |
| **Open Data**                   | Publicly funded systems should publish non‑sensitive data. | ESN includes scripts to generate redacted open data snapshots and dashboards. |
| **Record Retention**            | Data retention periods must comply with municipal policies.| Configurable retention; defaults to 2 years for reports, 1 year for images; custom policies supported. |
| **Procurement Transparency**    | Award criteria and performance metrics should be public.   | Corridor scores and service performance metrics are published in dashboards and reports. |
| **Localization**                | Software must support multiple languages.                 | ESN uses internationalization frameworks; currently supports English and Spanish, extensible to others. |
